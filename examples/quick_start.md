# ⚡ Quick Start: Primeros Pasos

Aquí aprendes a usar `research-frameworks` en 5 minutos.

---

## 1️⃣ Uso Básico (2 minutos)

### Opción A: Pregunta Simple
```
Investiga frameworks Python para dashboard analytics
```

**Qué recibis**: 
- Tabla con 5-6 opciones (Streamlit, Dash, Panel, etc)
- Pros/contras de cada una
- Recomendación según tu contexto

### Opción B: Pregunta Específica
```
Necesito elegir entre Streamlit vs Plotly Dash vs Bokeh
para dashboard de ventas con datos en tiempo real.
Equipo: 1 dev junior. Timeline: 1 mes.
```

**Qué recibís**:
- Comparativa profunda de las 3 opciones
- Árbol de decisión (¿prefieres facilidad o features?)
- 2 ejemplos de código
- Recomendación: Streamlit para timeline corto

## 2️⃣ Pasos Rápidos (5 minutos)

### A. Define tu caso

Piensa en:
- **¿QUÉ?** → Tipo de proyecto (API, frontend, DevOps, etc)
- **¿PARA QUÉ?** → Caso de uso específico
- **¿QUIÉN?** → Tamaño del equipo
- **¿CUÁNDO?** → Timeline

### B. Haz tu pregunta

Usa uno de estos formatos:

**Formato 1 - Directo:**
```
Investiga [categoría] para [caso]
```

Ejemplo:
```
Investiga herramientas de CI/CD para pipeline de 500+ builds/día
```

**Formato 2 - Con restricciones:**
```
Investiga [categoría] para [caso]
PERO [restricción 1]
PERO [restricción 2]
```

Ejemplo:
```
Investiga bases de datos NoSQL para startup
PERO presupuesto < $50/mes
PERO soporte en 30+ lenguajes
```

**Formato 3 - Comparativa directa:**
```
Compara [opción A] vs [opción B] vs [opción C]
para [caso específico]
```

Ejemplo:
```
Compara Postgres vs MongoDB vs DynamoDB
para app móvil con 100K usuarios
```

### C. Recibe tu análisis

Recibirás documento con:
1. **Executive Summary** (1 párrafo) - qué recomendar
2. **Tabla Comparativa** (estructura clara) - pros/contras
3. **Árbol de Decisión** (visual) - depende de TU contexto
4. **Ejemplos de Código** (2-3) - cómo empezar
5. **Roadmap** (semanas) - plan de adopción
6. **Referencias** (links) - donde aprender

## 4️⃣ Tips Avanzados

### Activar Investigación Extra Profunda

Añade `(DEEP)`:
```
Investiga (DEEP) landing page builders
para startup que necesita A/B testing avanzado
```

**Efecto**: Investigación más exhaustiva (90 min), 8+ opciones

### Personalizar Criterios de Evaluación

Especifica pesos:
```
Investiga Python testing frameworks
PRIORITIZA: velocidad (30%), community (40%), ease (30%)
IGNORE: ecosystem size
```

**Efecto**: Matriz ajustada a TUS valores

### Filtrar por Madurez

```
Investiga IA models para clasificación de texto
PERO solo frameworks con 3+ años en producción
PERO documentados en español/inglés
```

**Efecto**: Filtra opciones experimentales

## 5️⃣ Casos Donde FALLA (Evitar)

❌ **No uses para:**
- "¿Cuál es mejor, Python o JavaScript?" (muy genérico)
- "¿Por qué me da este error?" (debugging, no elección)
- "Enséñame a usar Django" (tutorial, no research)
- "¿Cuál es más rápido?" (necesita contexto)

✅ **Usa CUANDO:**
- Necesitas elegir entre 3+ opciones
- Tienes restricciones específicas
- Necesitas análisis estructurado
- Hay trade-offs que evaluar

---

## 📋 Checklist Antes de Preguntar

Antes de activar la Skill, pregúntate:

- [ ] ¿Es una decisión técnica real? (no teórica)
- [ ] ¿Tengo 3+ opciones para comparar?
- [ ] ¿Puedo describir mi caso en 2-3 oraciones?
- [ ] ¿Tengo restricciones claras? (timeline, presupuesto, team)
- [ ] ¿Necesito justificación basada en evidencia?

Si contestaste SÍ a 3+, ¡adelante!
